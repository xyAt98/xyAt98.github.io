today is 2023-12-26

想到什么就说什么吧， ubuntu默认的这个中文输入法好垃圾阿，吐槽一下子。

今年一整年把， 周围都是一种焦虑的氛围。还未有offer的时候，焦虑毕业去向；有了 offer但并不满意的时候，焦虑前途；每毕业的时候，焦虑毕业设计和毕业答辩；正式工作之后，焦虑裁员；裁员风波之后，还是焦虑裁员。这些所有的焦虑，归到一处都是生存问题罢了。二十多年的学生生涯，早就给我灌输了学历可以解决生存问题这一概念。这种大众念头为何诞生也挺好想的，上一辈读出来书的那些人基本都不愁工作问题。可中国发展得太快了，产业升级得太快了。刚改革开放的时候，百废俱兴，当然对知识和技能的需求量贼大，那个时候掌握知识的人也少。现在，一是产业升级了，原有产业要么趋于稳定不需要那么多人了，要么已经被替代了，整个产业都没了，跟这个产业相关的专业也当然会没落。而这种专业的没落在大学中是滞后于市场出现的。不过这不是我本篇日志想重点说的，乱七八糟七七八八的理由太多了，谁能总结得完。

我想着重说的只有一点，快速变化才是世界的主旋律。比如说我在面对学历和找工作这二者的相关性上曾根深蒂固的认为他俩一定是一个单调递增的关系。背后的原因是拿着过去二十多年的前人的经验去套现在的环境，二十年后世界早不一样了。这太正常了。曾几何时，最好的工作是外企，而后变成了大厂，如今变成了编制。这些变化甚至就发生在一两年内，我拿前一年学姐学长的经验去套如今的环境就可能不适用了。人一生几十年，世界能发生的事儿可太多了。这种时间上的宏大感和自身的渺小感是在我看历史相关书籍的时候体会尤为深刻。我想想阿，我想说什么呢？ 第一点是，我们在当下批判这个破环境的时候，我们今年找工作找不到学姐学长那样好的工作的时候，在抱怨运气何其不好的时候， 可能我们需要面对的现实是：这才是常态，有幸赶上红利期那才是何其幸运，那才是少数。拉到人生的长度来说，不幸是常态，幸运是罕见。不幸是本分？幸运是情分？（可以这么形容吗）生存从来都不容易，不要小看人生阿。将自己的生存主动权掌握在自己手上，不是依托于其他东西上，比如好的经济环境，好的学历。这句话用于自勉，我希望成为这样的人，成为抛开所有的外在标签之后仍然足够有生命力，足够鲜活的人。

那我想说明/ 想倡导的主题就是 ： 无为而治/等风来？ 大概是这个意思吧。活着就好，先活着把，遇到问题，解决问题。可以慢，不可以停止。

我还是很爱这个世界的，这个世界能让我阅读，就还挺好的。

可我还是睡不着，md嘤嘤嘤，再次吐槽以下，ubuntu的中文输入法烂到家了！

粘一句《希达多》的里的话：世界并非徐缓地行进在通向圆满之路；不，世界的每一瞬间皆为圆满。

概念

1. DOM

Document Object Model： 文件在计算机中的表现形式。

Linux的文件系统也是一个DOM树？

当一个浏览器加载一个页面的时候发生了什么？

浏览器先读取HTML文件， 将其加载成DOM树（个人感觉是描述了HTML中各种元素的关系，父子关系，兄弟关系什么的)。

浏览器还会加载CSS样式表（css可以定义在哪些地方， 应该有很多地方可以定义)，将样式给应用到对应的dom元素（节点)中。

最后应用JS代码，js代码最终应该会嵌入到HTML页面中？当对应行为发生时，触发js代码，页面上也会有相应的变化。（因此它是动态的)

CSS的一些关键概念

Cascading Style Sheets 层叠样式表, 个人理解， 是网页中的一个dom元素具有的样式是多个，层叠或许还相互冲突的，最终呈现在页面的样式是经过计算的最终的样式。

那么如何得到最终的样式呢？涉及到三个概念

  • 资源顺序
  • 优先级
  • 继承

1。 资源顺序

在优先级相同的情况下，展示后加载的资源样式

2。优先级

  • 选择器选择的越模糊优先级越低，以由上到下优先级依次递增：

    • 元素选择器（即span这种)

    • 类选择器（即在节点中定义了类名的那中classame=“xxx")

    • ID选择器（在节点中定义了ID id=“xxx’"，类名可以重复使用，id应该不能)

    • 内联样式（即style里定义的样式)

    • !important（在定义属性值后面家一个!important)

3. 继承

有些属性是默认是可以继承的， 有些属性则不能。可以指定一个属性是否要继承。

  • inherit
  • initial
  • unset

样式表的位置和级联层也会影响最终计算出来的CSS属性值。这里不再赘述。多用页面debug工具。
